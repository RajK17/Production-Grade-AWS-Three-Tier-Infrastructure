# Production-Grade AWS Three-Tier Infrastructure

## Key Features

* Multi-AZ deployment for high availability and fault tolerance
* Public and private subnet architecture within a custom VPC
* Application Load Balancer for traffic distribution
* Auto Scaling Groups with self-healing capabilities
* Amazon RDS MySQL Multi-AZ deployment with automated failover
* Secure access using IAM roles and AWS Systems Manager Session Manager
* Security Group chaining and Network ACL-based network protection
* CloudWatch monitoring, dashboards, and SNS alerting
* VPC Flow Logs for network visibility and auditing
* S3 lifecycle policies for storage optimization and cost reduction

## Architecture Components

* VPC
* Public & Private Subnets
* Internet Gateway
* NAT Gateway
* Application Load Balancer (ALB)
* Auto Scaling Group (ASG)
* EC2 Instances
* Amazon RDS MySQL (Multi-AZ)
* IAM Roles
* AWS Systems Manager (SSM)
* Amazon CloudWatch
* Amazon SNS
* Amazon S3
* VPC Flow Logs

## Project Goals

The primary objective of this project is to simulate a production-grade cloud environment that demonstrates high availability, scalability, security, monitoring, and operational excellence using AWS native services.

This repository contains the complete implementation guide, architecture diagrams, deployment steps, configuration details, and best practices used throughout the project.
